# Progress Log — 2026-05-21

## Summary
Continued development of TrendChannelV1 (long) and TrendChannelV2 (short) strategies.
Fixed several bugs in pattern detection, fill price logic, and channel state management.

## Bug Fixes

### 1. TP1 Fill Price — V2 (`strategies/trend_channel_v2.py`, `run_backtest_v2.py`, `tools/backtest_chart_v2.py`)
- **Before**: TP1 filled at candle close
- **After**: TP1 fills at the lower trendline touch price (`ch.lower_now`)
- Same override mechanism as SL: temporarily set `broker._current_prices` before placing MARKET order
- `tp1_price: current_lower` added to TP1 signal metadata

### 2. TP1 Fill Price — V1 (`strategies/trend_channel_v1.py`, `run_backtest.py`, `tools/backtest_chart.py`)
- **Before**: TP1 filled at candle close; trigger used fixed entry-time `self._tp1_price`
- **After**: TP1 triggers on current extended upper trendline (`ch.upper_now`), fills at that price
- Same `override_price` pattern applied in both backtest runner and chart tool

### 3. Stale Index Bug (`patterns/trend_line.py`) — Both `TrendLinePattern` and `TrendLinePatternUp`
- **Problem**: At channel lock time, `l2_idx` (H2/L2 index) is stored. Later when `_save_carry_from_locked()` is called, `_history` may have been trimmed, causing `_history[l2_idx]` to return a wrong candle's timestamp. This caused H1/L1 carry markers to appear "floating in the air" in the chart.
- **Fix**: Save H2/L2 timestamp directly at lock time in `_lck_l2_ts`. `_save_carry_from_locked()` now uses `self._lck_l2_ts` instead of index lookup.

### 4. Channel Freeze During Position (`patterns/trend_line.py`, `strategies/trend_channel_v1.py`, `strategies/trend_channel_v2.py`)
- **Problem**: While in a position, if price broke out of the channel, the pattern would unlock → carry → start detecting new channels. This caused H1 markers to update visually while in a trade, and TP1 to reference a different channel's trendline.
- **Fix**: Added `freeze()` / `unlock()` mechanism to both `TrendLinePattern` and `TrendLinePatternUp`.
  - `freeze()`: called on position entry fill — suppresses break-based unlock in `evaluate()`
  - `unlock()`: called on full position close — resets `_frozen`, saves carry, clears channel
  - V1: `freeze()` called in `on_fill(BUY)` (long entry)
  - V2: `freeze()` called in `on_fill(SELL)` (short entry)
