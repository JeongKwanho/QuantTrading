## 2. Strategy: TrendChannelV1

### Entry Condition

```
candle.low  <= lower_now          (touches lower channel line)
candle.close > lower_now          (closes inside channel)
candle.close < upper_now          (closes below upper channel line)
reward / risk >= min_rr (2.0)     (reward = upper_now - close, risk = close - low)
```

### Exit Conditions (priority order)

| Priority | Trigger | Action |
|----------|---------|--------|
| 1 | `candle.low <= sl_price` | Sell all ŌĆö market at sl_price |
| 2 | `candle.high >= h2_price` (after TP1) | Sell all ŌĆö market |
| 3 | `candle.high >= tp1_price` (before TP1) | Sell 50% ŌĆö market; move SL to avg_price |

- **SL v1**: `sl_price` = low of entry candle (tail end)
- **SL v2** (after TP1): `sl_price` = avg entry price (break-even)
- **TP1 target**: upper channel line at entry time (fixed)
- **TP2 target**: H2 price at entry time (fixed)
- **Cooldown**: after SL, no re-entry for N candles (default: 5)

## 3. Pattern: TrendLinePattern (Lock + Carry)

### Channel Detection (downtrend)

```
H1 = highest pivot high before L1
L1 = lowest pivot low in window
H2 = highest pivot high after L1
slope = (H2 - H1) / (H2.idx - H1.idx)  ŌåÆ must be < 0
gap = upper_at_L1 - L1.price
Containment check: all candles in [H1..H2] must stay inside
```

### Lock / Breakout Behavior

- Once a channel is detected, it is **locked** and extended forward one candle per tick
- **Breakout** = `candle.close > upper` or `candle.close < lower` (close-based, not high/low)
- On breakout or full position close (`unlock()`): H2 becomes the new H1 (**carry mechanism**)
- Carry prevents duplicate/continuous re-detection of the same trendline

### Retroactive Chart Fill

When a channel is first locked, the chart tool retroactively fills `upper`/`lower` values
from the H1 candle index backward using `upper_i = h1_price + slope * steps`.
This ensures channel lines visually pass through H1, H2, and L1.

## 4. Bugs Fixed

### Bug 1 ŌĆö Breakout used high/low instead of close

**Before**: channel breakout triggered when `candle.high > upper or candle.low < lower`  
**After**: `candle.close > upper or candle.close < lower`

### Bug 2 ŌĆö Entry allowed when close > upper_now

`_check_entry` was missing the `closed_inside = data.close < ch.upper_now` guard.
Entries above the upper channel line were incorrectly allowed.

### Bug 3 ŌĆö Channel fully reset on breakout (no carry)

After breakout, the pattern reset completely, leading to duplicate channels.  
**Fix**: on breakout or unlock, save H2 as `_carry_ts`/`_carry_price`. Next detection uses H2 as H1.

### Bug 4 ŌĆö SL fill at candle close instead of sl_price

SL orders were market orders filling at `candle.close`. Should fill at `entry_candle.low` (the tail).  
**Wrong fix attempt**: set `price=sl_price` on the SL signal ŌåÆ `Signal.order_type` auto-derives
`LIMIT` when `price is not None` ŌåÆ order never fills ŌåÆ `on_fill` never called ŌåÆ
`_position_qty` accumulates across trades ŌåÆ 96% balance drain.  
**Correct fix**: keep SL signal as `price=None` (MARKET). In backtest loop, temporarily set
`broker._current_prices[sym] = sl_price` before placing the order, then restore to `candle.close`.

### Bug 5 ŌĆö `_position_qty` not accumulating after on_fill mismatch (from Bug 4)

Root cause was Bug 4. Once SL fills correctly via `on_fill`, `_position_qty` is reset to 0
and subsequent BUY entries accumulate from 0 as expected.
