## 1. System Design

### Decisions Made

| Item | Decision |
|------|----------|
| Language | Python 3.x |
| Exchange | Binance Futures (USDT-M) only — extensible via BaseBroker |
| API Framework | FastAPI + uvicorn |
| Database | PostgreSQL (persistent) + Redis (cache) — deferred, using in-memory mock for now |
| Frontend | React 18 + TypeScript (not started yet) |
| Authentication | API Key via `X-API-Key` header |

### Architecture Principles

- Strategies return `Signal` objects only — no direct broker access
- All broker implementations share `BaseBroker` interface — swap exchange by changing `.env`
- Frontend communicates only through `backend/api/` via REST + WebSocket
- `backtest/` uses `broker/mock.py` — never the live broker
- Folder isolation: each layer is worked on independently

### Key Files Explained

**`backend/broker/base.py`**
- `Signal` — strategy output: direction (BUY/SELL), quantity, price (None = market order)
- `BaseBroker` — abstract class with `place_order`, `cancel_order`, `get_order`, `get_open_orders`, `get_balances`, `get_positions`, `get_ticker`, `get_orderbook`, `get_ohlcv`
- Callback hooks: `set_on_fill()`, `set_on_ticker()` — engine registers these to receive real-time events

**`backend/broker/binance.py`**
- Implements all `BaseBroker` methods using `python-binance` `AsyncClient`
- `connect()` creates `AsyncClient` + `BinanceSocketManager`
- `subscribe_ticker()` / `subscribe_user_stream()` — WebSocket streams for price and fill events
- `_parse_order()` — converts raw Binance response to `Order` dataclass

**`backend/api/state.py`**
- `AppState` holds the broker instance
- Falls back to mock data when broker is not connected (no `.env` set)
- Designed for easy swap: replace mock methods with real engine calls later

**`backend/api/main.py`**
- `lifespan` context manager: connects broker on startup, disconnects on shutdown
- Runs with mock data if `BINANCE_API_KEY` is not set
- Swagger UI available at `/docs`

## 6. Session 2 — Multi-Timeframe Strategy System

### New Files

| File | Description |
|------|-------------|
| `strategies/base.py` | `BaseStrategy`, `MarketData`, `Signal`, `FillEvent` dataclasses |
| `strategies/layers.py` | `TimeframeLayer` — AND logic across strategies on one timeframe |
| `strategies/groups.py` | `ScalpingGroup` (3tf), `SwingGroup` (2tf), `PositionGroup` (1tf) + `build_group()` |
| `broker/mock.py` | `MockBroker` — leverage, fees, liquidation, limit order fill simulation |
| `backtest/historical.py` | Historical backtest — paginated REST fetch, multi-timeframe |
| `backtest/live_paper.py` | Live paper trading — concurrent WebSocket streams per timeframe |
| `backtest/report.py` | `BacktestReport` — return, drawdown, Sharpe, win rate, profit factor |

### Architecture: Multi-Timeframe Filtering

```
ScalpingGroup  (3 timeframes)
  large  (1d / 1w / 3d)   → TimeframeLayer → all strategies agree? → confirmed_direction
  medium (4h / 6h / 8h / 12h) → same
  small  (1m-1h)           → same → if all 3 confirmed & same direction → Signal

SwingGroup     (2 timeframes)
  large  → confirmed?
  medium → confirmed & matches large? → Signal

PositionGroup  (1 timeframe)
  large  → confirmed? → Signal
```

### Key Design Decisions

- `TimeframeLayer.evaluate()` — if all strategies return the same direction, confirms it; any disagreement clears to `None`
- Previous confirmed direction is preserved until overridden (sticky state)
- `BaseGroup.primary_timeframe` — smallest (fastest) timeframe, used for equity curve recording
- `build_group(config, strategy_registry)` — converts frontend JSON config → Python group instance
- `all_strategies()` — returns flat list of all strategy instances for `on_fill` callback wiring

### MockBroker Details

- Taker fee: 0.04% (market orders), Maker fee: 0.02% (limit orders)
- Liquidation price formula: `entry × (1 − 1/leverage + 0.4%)`
- `update_price()` checks liquidation on each tick
- `check_limit_orders(high, low)` fills pending limit orders within candle range
- `reset()` clears all state for clean backtest reruns

### Backtest Engines

**HistoricalBacktest**
- Fetches all required timeframes in parallel (`asyncio.gather`)
- Merges candles by timestamp, sorts large→small within same timestamp
- Feeds each candle to `group.on_candle(timeframe, data)`
- Records equity curve on primary timeframe candles only

**LivePaperBacktest**
- Opens one WebSocket stream per timeframe with `asyncio.gather`
- Uses `asyncio.Lock` to prevent concurrent broker state mutation
- Unconfirmed candles only update price (primary timeframe only)
- `on_fill` callback wired to all strategies in the group
