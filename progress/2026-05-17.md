### Key Files Explained

**`backend/broker/base.py`**
- `Signal` ‚Äî strategy output: direction (BUY/SELL), quantity, price (None = market order)
- `BaseBroker` ‚Äî abstract class with `place_order`, `cancel_order`, `get_order`, `get_open_orders`, `get_balances`, `get_positions`, `get_ticker`, `get_orderbook`, `get_ohlcv`
- Callback hooks: `set_on_fill()`, `set_on_ticker()` ‚Äî engine registers these to receive real-time events

**`backend/broker/binance.py`**
- Implements all `BaseBroker` methods using `python-binance` `AsyncClient`
- `connect()` creates `AsyncClient` + `BinanceSocketManager`
- `subscribe_ticker()` / `subscribe_user_stream()` ‚Äî WebSocket streams for price and fill events
- `_parse_order()` ‚Äî converts raw Binance response to `Order` dataclass

**`backend/api/state.py`**
- `AppState` holds the broker instance
- Falls back to mock data when broker is not connected (no `.env` set)
- Designed for easy swap: replace mock methods with real engine calls later

**`backend/api/main.py`**
- `lifespan` context manager: connects broker on startup, disconnects on shutdown
- Runs with mock data if `BINANCE_API_KEY` is not set
- Swagger UI available at `/docs`

## 6. Session 2 ‚Äî Multi-Timeframe Strategy System

### New Files

| File | Description |
|------|-------------|
| `strategies/base.py` | `BaseStrategy`, `MarketData`, `Signal`, `FillEvent` dataclasses |
| `strategies/layers.py` | `TimeframeLayer` ‚Äî AND logic across strategies on one timeframe |
| `strategies/groups.py` | `ScalpingGroup` (3tf), `SwingGroup` (2tf), `PositionGroup` (1tf) + `build_group()` |
| `broker/mock.py` | `MockBroker` ‚Äî leverage, fees, liquidation, limit order fill simulation |
| `backtest/historical.py` | Historical backtest ‚Äî paginated REST fetch, multi-timeframe |
| `backtest/live_paper.py` | Live paper trading ‚Äî concurrent WebSocket streams per timeframe |
| `backtest/report.py` | `BacktestReport` ‚Äî return, drawdown, Sharpe, win rate, profit factor |

### Architecture: Multi-Timeframe Filtering

```
ScalpingGroup  (3 timeframes)
  large  (1d / 1w / 3d)   ‚Üí TimeframeLayer ‚Üí all strategies agree? ‚Üí confirmed_direction
  medium (4h / 6h / 8h / 12h) ‚Üí same
  small  (1m-1h)           ‚Üí same ‚Üí if all 3 confirmed & same direction ‚Üí Signal

SwingGroup     (2 timeframes)
  large  ‚Üí confirmed?
  medium ‚Üí confirmed & matches large? ‚Üí Signal

PositionGroup  (1 timeframe)
  large  ‚Üí confirmed? ‚Üí Signal
```

### Key Design Decisions

- `TimeframeLayer.evaluate()` ‚Äî if all strategies return the same direction, confirms it; any disagreement clears to `None`
- Previous confirmed direction is preserved until overridden (sticky state)
- `BaseGroup.primary_timeframe` ‚Äî smallest (fastest) timeframe, used for equity curve recording
- `build_group(config, strategy_registry)` ‚Äî converts frontend JSON config ‚Üí Python group instance
- `all_strategies()` ‚Äî returns flat list of all strategy instances for `on_fill` callback wiring

### MockBroker Details

- Taker fee: 0.04% (market orders), Maker fee: 0.02% (limit orders)
- Liquidation price formula: `entry √ó (1 ‚àí 1/leverage + 0.4%)`
- `update_price()` checks liquidation on each tick
- `check_limit_orders(high, low)` fills pending limit orders within candle range
- `reset()` clears all state for clean backtest reruns

### Backtest Engines

**HistoricalBacktest**
- Fetches all required timeframes in parallel (`asyncio.gather`)
- Merges candles by timestamp, sorts large‚Üísmall within same timestamp
- Feeds each candle to `group.on_candle(timeframe, data)`
- Records equity curve on primary timeframe candles only

**LivePaperBacktest**
- Opens one WebSocket stream per timeframe with `asyncio.gather`
- Uses `asyncio.Lock` to prevent concurrent broker state mutation
- Unconfirmed candles only update price (primary timeframe only)
- `on_fill` callback wired to all strategies in the group

## Session 3 ‚Äî Pattern Detection Framework (`patterns/`)

### Goal

Before writing concrete strategies, separate reusable "detection building blocks" into a dedicated `patterns/` folder. Strategies will import and combine patterns like lego pieces.

### New Files

| File | Description |
|------|-------------|
| `patterns/__init__.py` | Empty package init |
| `patterns/base.py` | `BasePattern` abstract class + `PatternResult` dataclass |
| `patterns/trend_line.py` | `TrendLinePattern` ‚Äî uptrend and downtrend channel detection |

### `PatternResult` dataclass

```python
@dataclass
class PatternResult:
    detected:  bool
    direction: Literal["BUY", "SELL"] | None
    strength:  float   # 0.0 ~ 1.0
    name:      str
```

### `TrendChannel` dataclass

Stores the two parallel trendlines computed from 3 pivot points.

```
direction = "up":
    lower line  ‚îÄ‚îÄ‚îÄ L1 ‚Üí L2  (two lowest pivot lows)
    upper line  ‚îÄ ‚îÄ parallel, passes through H1 (highest pivot high)

direction = "down":
    upper line  ‚îÄ‚îÄ‚îÄ H1 ‚Üí H2  (two highest pivot highs)
    lower line  ‚îÄ ‚îÄ parallel, passes through L1 (lowest pivot low)
```

### Pivot Detection (`_find_pivots`)

For each candle `i` in the last `window` candles:
- A **pivot low** at `i`: `low[i]` is strictly less than all neighbors in `[i-k, i+k]`
- A **pivot high** at `i`: `high[i]` is strictly greater than all neighbors in `[i-k, i+k]`

Parameters: `window=50`, `pivot_k=2` (configurable).

### Uptrend Channel Algorithm

1. Find H1 = highest pivot high in window
2. L1 = lowest pivot low **before** H1 (X-axis: L1 < H1)
3. L2 = lowest pivot low **after** H1 (X-axis: H1 < L2)
4. `slope = (L2.low - L1.low) / (L2.idx - L1.idx)` ‚Äî must be **> 0**
5. `gap = H1.high - lower_trendline_at(H1)` ‚Äî channel width
6. **Containment check**: every candle in [L1, L2] must satisfy `low ‚â• lower(i)` and `high ‚â§ upper(i)` ‚Äî if any candle breaks out, channel is rejected

### Downtrend Channel Algorithm

1. Find L1 = lowest pivot low in window
2. H1 = highest pivot high **before** L1 (X-axis: H1 < L1)
3. H2 = highest pivot high **after** L1 (X-axis: L1 < H2)
4. `slope = (H2.high - H1.high) / (H2.idx - H1.idx)` ‚Äî must be **< 0**
5. `gap = upper_trendline_at(L1) - L1.low`
6. **Containment check**: every candle in [H1, H2] must satisfy `high ‚â§ upper(i)` and `low ‚â• lower(i)`
