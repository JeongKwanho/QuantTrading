The FVG work went through several definitions:

- Bullish/bearish candle-color filtering was tested and then removed.
- A stricter bullish FVG structure was tested:
  - connect two pivot highs with negative slope,
  - find a horizontal liquidity low below that line,

Decision: **FVG was paused for now**. The files remain in the workspace for later reference.

## 2. TrendChannelV1 Analysis

Reviewed `strategies/trend_channel_v1.py`.

Current strategy behavior:

- Long-only downtrend-channel lower-bound bounce.
- Entry:
  - candle low touches or pierces `lower_now`,
  - candle closes back above `lower_now`,
  - candle closes below `upper_now`,
  - reward/risk to channel upper line is at least `min_rr`.
- Exit:
  - SL first,
  - TP2 after TP1,
  - TP1 partial exit if channel upper is reached.

Key issues identified:

- Entry is too simple and often catches falling price.
- Strategy state is set to in-position before actual fill.
- OHLC backtests cannot know whether SL or TP happened first inside a candle.
- Group/layer architecture can lose strategy-level metadata like `sl`, `tp1`, `tp2`.

## 3. Experimental Improved Strategy

Added:

```text
strategies/decrease_trend_channer_v1.py
```

This started as a clean copy of `TrendChannelV1`, then added filters while keeping the original TP/SL structure.

Added improvements:

### Channel Quality Filter

- Minimum channel bars.
- Channel width in ATR range.
- Minimum slope normalized by ATR.
- Minimum down-leg size normalized by ATR.

### Touch Quality Filter

- Minimum lower-wick ratio.
- Maximum lower-line pierce depth in ATR.
- Minimum recovery above `lower_now` in ATR.

### Market Environment Filter

- RSI cap.
- Minimum volume ratio versus previous average volume.
- Optional `require_bull_market`.

Excluded intentionally:

- No next-candle confirmation entry yet.
- No TP/SL structure redesign yet.

## 6. Bull-Market Filter Experiments

Tested the idea that this setup may work better as:

```text
bull trend pullback channel long
```

instead of pure downtrend-channel reversal.

Tested filters:

- SMA50 > SMA200 and close > SMA200.
- SMA20 > SMA100 and close > SMA100.
- SMA5 > SMA20 and close > SMA20.
- Recent bull-state grace window.

Result:

- Strict MA filters removed almost all entries.
- A broad recent-bull grace window became too permissive and matched unfiltered results.
- Conclusion: simple MA filters are not enough to define the intended "bull-market pullback" context.

Better future direction:

```text
Use higher-timeframe TrendLinePatternUp / bullish structure confirmation
instead of simple moving averages.
```

## 7. Realtime-Oriented Strategy

Added:

```text
strategies/decrease_trend_channer_live_v1.py
```

Class:

```python
DecreaseTrendChannerLiveV1
```

Purpose:

- A live-safe version of the New+B idea.
- Keeps recent bull-market context internally.
- Uses the improved channel/touch/environment filters.

Realtime safety changes:

- Does not set `_in_position = True` until a BUY fill arrives.
- Does not complete TP/SL state changes until SELL fills arrive.
- Uses close-based exits to avoid ambiguous intrabar OHLC ordering.
- Blocks duplicate signals while entry/exit fill is pending.

Default bull context:

```text
SMA5 > SMA20
close > SMA20
recent bull-state grace = 50 bars
```

Note:

- This is intended for standalone strategy execution or a group architecture that preserves strategy metadata.
- Current `TimeframeLayer` / `Group` logic rebuilds signals and can discard metadata like `sl`, `tp1`, and `tp2`.
