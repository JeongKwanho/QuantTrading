# Progress - 2026-05-26

## Summary

Updated **OBChannelV1** so partial take-profit and second target behavior match the intended liquidity-based exit model, then added a stricter structure-based downtrend filter for bullish OB detection.

The strategy now avoids more OB candidates that appear during shallow pullbacks in an uptrend, without using a slow EMA slope filter.

---

## 1. Restored TP1 / TP2 / SL2 Exit Flow

Updated `strategies/ob_channel_v1.py`.

### TP1

- Previous diagnostic state sold 100% at `tp1_price`.
- Changed TP1 to sell only **50% of the current position**.
- After TP1, remaining position stays open.
- `new_sl` metadata now records `avg_price`.

### TP2

- TP2 was re-enabled.
- TP2 now exits the remaining position at a market order when price reaches the previous OB swing high liquidity area.
- Instead of using only the max high of the prior 7 candles, TP2 now prefers the **previous pivot high before the OB candle**.
- If the initial TP2 is below or equal to TP1, the strategy searches for a previous pivot high above TP1 to avoid the old issue where TP2 could fill below TP1.

### SL2

- After TP1, SL2 exits the remaining position at `avg_price`.
- After TP1, exit priority is:

```text
TP2 first
SL2 second
```

## 2. Added Structure-Based Downtrend Filter

Updated `patterns/order_block.py`.

### Problem

The previous downtrend filter used:

```text
last pivot high < previous pivot high
AND
last pivot low < previous pivot low
```

This could still detect OBs during shallow pullbacks inside a larger uptrend.

### New Filter

Added a stricter OB-candidate filter:

- Reject OBs when recent structure is clear `HH + HL`.
- Require the OB candle to appear after a previous pivot high.
- Require price to have dropped at least `min_drop_atr` ATR from that pivot high.
- Require the OB candle close to sit low enough inside the down leg, controlled by `max_ob_range_pos`.

Current defaults:

```python
min_drop_atr = 1.0
max_ob_range_pos = 0.65
```

No EMA slope filter was added because EMA50 can react too slowly for this setup.
