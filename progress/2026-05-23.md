# Progress — 2026-05-23

## Summary
Identified and fixed two fundamental logic bugs in **OBChannelV1**, and ran a temporary TP1-only test to isolate remaining issues.

---

## 1. Bug Fix — Entry Timing (`strategies/ob_channel_v1.py`)

### Problem
OB detection fires on the **engulfing candle** (the bullish candle that confirms the OB pattern). The previous code allowed entry on that same candle, which is impossible in practice — the engulfing candle IS the confirmation candle, so the earliest possible entry is the candle after it.

### Fix
Added `_ob_detected_ts` to store the engulfing candle's timestamp. `_check_first_entry` now skips entry if the current candle matches `_ob_detected_ts`, while still updating `_max_high_since_ob` so TP1 tracking includes the engulfing candle's high.

```python
# _on_new_ob
self._ob_detected_ts = data.timestamp   # engulfing candle — no entry here

# _check_first_entry
self._max_high_since_ob = max(self._max_high_since_ob, data.high)
if data.timestamp == self._ob_detected_ts:
    return []
```

**Effect**: Entry count dropped significantly (e.g., 1d: 72 → 40), removing false early entries.

---

## 2. Bug Fix — TP1 Definition

### Problem
Because entry was firing on the engulfing candle itself, `tp1_price` was being set to the engulfing candle's single high — not the intended max high over a range of candles.

### Correct Logic
- TP1 tracking starts at the engulfing candle's high (set in `_on_new_ob`)
- Updated every candle via `_check_first_entry` until entry fires
- `tp1_price` = **max high (wick included) from engulfing candle to 1st entry candle**

With the entry timing fix in place, this now works correctly.
